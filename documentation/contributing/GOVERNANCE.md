# Project Owners

## Description

We built Hylozoa from the ground up and made it into what it is today.
We own the project.

## Rights

1. Accept or deny any issues, PR or contributing in any kind

## Duties

1. Act fair
2. Act selflessly
3. Act in Hylozoa's best interest
4. Ensure everyone has access to the latest release of Hylozoa for no money price
5. Ensure everyone has acess to updated documentation
6. Ensure the builds are working properly
7. Ensure the community has discussion space:
  - Join the discord: Link to be published later.
  - Follow the reddit: Link later.
  - Repo Issues
8. Ensure the quality of the community's work environment
9. "Rent" project space to the contributors
  - Branches
  - Issues
10. Accept or deny PRs

# Reviewers

## Description

Community member that look at issues, branches, PRs and accept or deny them.  
Do the first quality control.

## Rights

1. Be appreciated
2. Inherits contributors' rights
3. Open _(but not merge)_ pull requests
4. Close issues and pull requests. A reason why must be provided.

## Duties

1. Inherits contributors' duties
2. Sanitarize issues and pull requests

-------
# Contributors

## Description

Non-project owners who offer their free time and effort to help the project grow.

## Rights

1. Be appreciated
2. Open issues
3. Create new branches
4. "Rent" a branch
  - A branch is owned by a user when:
    - It is created
    - It is claimed in the event it stood idle for 30 days. Idle meaning no push or discussion by anyone.
  - Multiple contributors can "Rent" a single branch
    -  Explicit rights should be granted in saying/texting within the community's discussion spaces.  
5. Commit code in rented branchs other than `main` and `dev`
6. Fork any branch
7. Fork the project
8. Comment issues or pull requests
9. Request a Review
10. Use AI as a tool
11. Be welcomed into the community's discussion spaces
12. Stop contributing anytime 

## Duties

1. Be kind to other members and owners
2. No bullying or NSFW
3. Report bullying or other members misconduct
4. Ensure the quality of contributions
  - AI slop will not be accepted so make sure you are familiar with your work
5. Work within an issue
6. Answer the project owners' community calls
